workflow_engine.mixins package¶

These mixins abstract out common fields across workflow engine models.

Several workflow engine models use GenericForeignKey from Django’s Content Types Framework. The use of mixins improves consistency and reduces code duplication in the associated models that need Generic Relations.

workflow_engine.mixins.stateful module¶

class workflow_engine.mixins.stateful.Stateful(*args, **kwargs)[source]¶

Bases: django.db.models.base.Model

Provides an object_state field using the Django FSM package.

Object states can be used for various purposes.

For example, they may indicate that an object is in or has passed (or not passed) a section of a workflow. (for example Pending, Processing or Done).

They can also be used to distinguish objects at a coarse level (for example PASSED, FAILED).

When used with a workflow_engine.strategies.wait_strategy.WaitStrategy they can be used to indicate when the object can proceed to the next workflow node (for example MANUAL_QC, INCOMPLETE)
